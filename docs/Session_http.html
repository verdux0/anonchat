Sesiones Web: Mecanismos, Implementación y Seguridad

Introducción a las Sesiones Web

Una sesión web es el mecanismo fundamental que permite a las aplicaciones web mantener información de estado sobre las interacciones de un usuario durante su visita. A diferencia del protocolo HTTP que es inherentemente sin estado, las sesiones proporcionan continuidad y personalización.

Propósito Principal

  • Persistencia de estado: Recordar información del usuario entre múltiples páginas o visitas

  • Experiencias personalizadas: Mantener preferencias, configuraciones y datos específicos

  • Funcionalidades críticas: Habilitar carritos de compra, autenticación y seguimiento de actividad

Mecanismo de Funcionamiento

1. Inicio y Autenticación

// Ejemplo de autenticación asíncrona
const response = await fetch('login_api.php', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ usuario: 'ejemplo', clave: '123456' })
});

2. Generación del ID de Sesión Único

  • El servidor genera un identificador único tras autenticación exitosa

  • Este ID es la llave para acceder a los datos de sesión del usuario

3. Almacenamiento en Cookies HTTP-only

// PHP maneja automáticamente la cookie de sesión
session_start(); // Crea/recupera sesión y envía cookie al
cliente

Características de la Cookie de Sesión:

  • ✅ HttpOnly: Inaccesible desde JavaScript (protección contra XSS)

  • ✅ Segura: Se transmite solo sobre HTTPS en entornos productivos

  • ✅ Temporal: Expira al cerrar el navegador o después de inactividad

4. Mantenimiento del Estado

// En cada página protegida
session_start();
if (!isset($_SESSION['autenticado']) ||
$_SESSION['autenticado'] !== true) {
    header('Location: login.php');
    exit;
}

5. Terminación Controlada

// logout.php - Eliminación completa de la sesión
session_start();
$_SESSION = array(); // Limpiar variables de sesión

// Eliminar cookie del cliente
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
    $params["path"], $params["domain"]);
}

session_destroy(); // Destruir sesión en el servidor

6. Configuración de sesión segura

ini_set('session.cookie_httponly', 1); 
ini_set('session.cookie_secure', 1); 
// Usar solo en producción con HTTPS 
ini_set('session.use_strict_mode', 1); 
ini_set('session.cookie_samesite', 'Strict'); 
ini_set('session.gc_maxlifetime', 1800); // 30 minutos

ini_set() sirve para modificar configuraciones de PHP en tiempo de ejecución.
Esas configuraciones normalmente están en el archivo php.ini, pero con ini_set() puedes ajustarlas desde el código.

En tu caso, estás configurando cómo se manejan las sesiones:

🔹 ini_set('session.cookie_httponly', 1);

Hace que la cookie de sesión solo pueda ser leída por el servidor, no por JavaScript.
✔️ Previene ataques XSS.

🔹 ini_set('session.cookie_secure', 1);

Obliga a que la cookie de sesión solo se envíe por HTTPS.
⚠️ Debe usarse solo si el sitio está en HTTPS, porque si no la sesión se pierde.

🔹 ini_set('session.use_strict_mode', 1);

Evita que PHP acepte IDs de sesión inválidos o creados manualmente.
✔️ Protege contra ataques de “session fixation”.

🔹 ini_set('session.cookie_samesite', 'Strict');

Evita que la cookie se envíe en peticiones externas.
✔️ Protege contra CSRF.
Modo Strict es el más seguro.

🔹 ini_set('session.gc_maxlifetime', 1800);

Define cuánto tiempo los datos de sesión se conservan antes de que el garbage collector los elimine.
➡️ Aquí: 30 minutos.

Comprobar valores:

  1. Crea un archivo llamado por ejemplo info.php
  2. Pon dentro:
`<?php  phpinfo();`
  1. Ábrelo en tu navegador:

http://tusitio/info.php

  1. Busca la sección session → ahí verás todas las directivas tal como están configuradas.

✔️ Es la forma más completa ✔️ Te muestra valores locales y globales
⚠️ No olvides ELIMINAR ese archivo después (es un riesgo de seguridad
si lo dejas público).

Opción 2: Usar ini_get() para ver solo lo que te interesa**

3. 📜 JavaScript con Fetch (app.js)

document.getElementById('loginForm').addEventListener('submit', async function(e) {
    e.preventDefault();
    
    const user = document.getElementById('user').value;
    const password = document.getElementById('password').value;
    const messageDiv = document.getElementById('message');
    
    messageDiv.textContent = 'Verificando...';
    
    try {
        const response = await fetch('login.php', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify({ user, password })
        });
        
        const result = await response.json();
        
        if (result.success) {
            messageDiv.textContent = '✅ Login correcto! Redirigiendo...';
            setTimeout(() => {
                window.location.href = 'welcome.php';
            }, 1000);
        } else {
            messageDiv.textContent = '❌ ' + result.message;
        }
        
    } catch (error) {
        messageDiv.textContent = '❌ Error de conexión';
        console.error('Error:', error);
    }
});

4. 🔐 PHP de Login (login.php)

<?php
session_start();
header('Content-Type: application/json');

// Configuración simple
$host = 'localhost';
$db   = 'login';
$user = 'root';
$pass = '';

try {
    $pdo = new PDO("mysql:host=$host;dbname=$db", $user, $pass);
} catch (PDOException $e) {
    echo json_encode(['success' => false, 'message' => 'Error BD']);
    exit;
}

// Obtener datos JSON
$input = file_get_contents('php://input');
$data = json_decode($input, true);

$user = $data['user'] ?? '';
$password = $data['password'] ?? '';

// Buscar usuario
$stmt = $pdo->prepare("SELECT * FROM usuarios WHERE user = ?");
$stmt->execute([$user]);
$usuario = $stmt->fetch();

if ($usuario && password_verify($password, $usuario['password'])) {
    $_SESSION['user'] = $user;
    echo json_encode(['success' => true]);
} else {
    echo json_encode(['success' => false, 'message' => 'Usuario o contraseña incorrectos']);
}
?>

6. 🚪 Logout Completo (logout.php)

<?php
session_start();

// 1. Limpiar variables de sesión
$_SESSION = [];

// 2. Eliminar cookie de sesión del cliente
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(
        session_name(), 
        '', 
        time() - 42000,
        $params["path"], 
        $params["domain"], 
        $params["secure"], 
        $params["httponly"]
    );
}

// 3. Destruir sesión en el servidor
session_destroy();

// 4. Redirigir al login
header('Location: index.html');
exit;
?>

✅ 1. session_start();

Activa la sesión actual o crea una si no existía.

Aunque vayas a destruirla, primero debes iniciarla para poder manipularla.

✅ 2. Limpiar variables de sesión

$_SESSION = [];

Esto borra todas las variables almacenadas en la sesión, pero la sesión en sí aún sigue existiendo.

Ejemplo: si había $_SESSION['usuario'], ahora ya no existe.

✅ 3. Eliminar la cookie de sesión del cliente

if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(
        session_name(), 
        '', 
        time() - 42000,
        $params["path"], 
        $params["domain"], 
        $params["secure"], 
        $params["httponly"]
    );
}

¿Qué hace esto?

🔹 session.use_cookies

Comprueba si PHP usa cookies para almacenar el ID de sesión (casi siempre es sí).

🔹 session_get_cookie_params()

Obtiene las configuraciones actuales de la cookie de sesión:

  • path

  • domain

  • secure

  • httponly

  • samesite (en versiones nuevas)

Las usa para borrar la cookie con los mismos parámetros.

🔹 setcookie(..., time() - 42000, ...)

Crea una cookie con fecha de expiración en el pasado → esto elimina la cookie del navegador.

📌 Sin esto, la sesión podría seguir activa desde el lado del cliente aunque la destruyas en el servidor.

✅ 4. Destruir la sesión en el servidor

session_destroy();

Esto borra la sesión almacenada en el servidor (el archivo de sesión).
Ahora ya no existe ni el ID ni las variables.

✔️ Aquí termina completamente la sesión.

✅ 5. Redirigir al login

header('Location: index.html');
exit;

Envía al usuario de vuelta a la página de inicio de sesión.

exit evita que se siga ejecutando código después de la redirección.
